Composition & Structure

Define videos, layer elements, and control timing
<Composition> Used
Defines a video's metadata: dimensions, FPS, duration, and default props. The entry point for every Remotion video. Registered in Root.tsx.
<Composition id="ReleaseClip" width={1080} height={1920} fps={30} durationInFrames={750} // 25 sec component={ReleaseClip} defaultProps={defaultProps} />
<AbsoluteFill> Used
A full-size, absolutely positioned div. The fundamental building block for layering elements like in a video editor timeline. Lower in the tree = higher in the stack.
Layer stack (bottom to top): +-------------------------------+ | AbsoluteFill: VideoBackground | z=0 +-------------------------------+ | AbsoluteFill: FilmGrain | z=1 +-------------------------------+ | AbsoluteFill: TextOverlay | z=2 +-------------------------------+ | AbsoluteFill: CTABar | z=3 (on top) +-------------------------------+
<Sequence> Used
Time-shifts its children. Use from to delay appearance and durationInFrames to limit visibility. Inside a Sequence, useCurrentFrame() resets to 0 at the start.
Timeline (frames 0-750): |===HOOK (0-90)=====| |========BUILD (90-240)========| |==DROP (240-360)==| |=====ENDCARD (600-750)====| // Each is a <Sequence from={X} durationInFrames={Y}>
<Series> Available
Displays scenes one after another automatically. Each <Series.Sequence> starts where the previous one ends. No manual from math needed.
<Series> <Series.Sequence durationInFrames={90}> {/* Hook scene: frames 0-90 */} </Series.Sequence> <Series.Sequence durationInFrames={150}> {/* Build scene: frames 90-240 */} </Series.Sequence> </Series>
<Freeze> Available
Freezes all children at a specific frame. Useful for holding a video frame still (e.g., pause on a DJ reaction shot) while text animates on top.
<Freeze frame={30}> {/* Everything inside sees frame 30 forever */} <OffthreadVideo src={src} /> </Freeze>
<Loop> Available
Repeats its children. Great for looping background animations, particle effects, or repeating beat-synced pulses without duplicating code.
<Loop durationInFrames={15}> {/* This 15-frame animation loops forever */} <PulseEffect /> </Loop>

Animation Primitives

Frame-driven hooks and functions that power every motion
useCurrentFrame() Used
Returns the current frame number. THE animation driver in Remotion. Every animation must derive from this hook (never CSS transitions or requestAnimationFrame).
const frame = useCurrentFrame(); // frame = 0, 1, 2, ... up to durationInFrames // Inside a <Sequence from={90}>: // frame resets to 0 at the sequence start
useVideoConfig() Used
Returns { fps, width, height, durationInFrames, id } from the current Composition. Essential for fps-aware spring() animations.
const { fps, width, height, durationInFrames } = useVideoConfig(); // fps = 30, width = 1080, height = 1920
interpolate() Used
Maps a value from one range to another. The workhorse of Remotion animation. Supports clamping via extrapolateLeft / extrapolateRight and custom Easing curves.
How interpolate() works: Input range: [0 -------- 30 -------- 60] | | Output range: [0.0 ----- 0.5 ------- 1.0] Frame 0 -> opacity 0.0 (invisible) Frame 30 -> opacity 0.5 (half visible) Frame 60 -> opacity 1.0 (fully visible) interpolate(frame, [0, 60], [0, 1], { extrapolateRight: "clamp" }) With Easing: interpolate(frame, [0, 60], [0, 1], { easing: Easing.bezier(0.8, 0.22, 0.96, 0.65), extrapolateLeft: "clamp", extrapolateRight: "clamp", })
spring() Used
Physics-based animation that goes from 0 to 1 with natural bounce/settle. Configure mass, damping, stiffness, and overshootClamping. Visual editor at springs.remotion.dev.
spring() output over time: 1.1 | *** 1.0 | ** ***************************** <-- settles at 1.0 0.8 | * 0.6 | * 0.4 | * 0.2 | * overshoot (bouncy!) 0.0 |*** +-----|---------|---------|--- f=0 f=5 f=10 f=20 spring({ frame, fps, config: { damping: 10, // low = more bounce stiffness: 100, // high = faster snap mass: 1, // heavy = slower }}) Our presets: CTABar: damping=200 (no bounce, smooth slide) EndCard: damping=15, stiff=120 (subtle bounce) SlamIn: damping=12, stiff=200 (snappy punch)
interpolateColors() Available
Smoothly transition between colors over time. Accepts hex, rgb, rgba, hsl, and CSS named colors. Perfect for genre-based color washes.
const color = interpolateColors( frame, [0, 60], ["#f0654a", "#f5b731"] // coral -> amber ); // Returns "rgba(242, 140, 60, 1)" at frame 30
Easing Available
Custom easing curves for interpolate(). Includes linear, ease, bezier(), bounce, elastic(), back(), and more. Combinable with Easing.in() / out() / inOut().
import { Easing } from "remotion"; interpolate(frame, [0, 30], [0, 1], { easing: Easing.inOut(Easing.ease), }) // Pre-built: linear, quad, cubic, sin, // circle, exp, bounce, elastic(), back()
measureSpring() Available
Calculate exactly how many frames a spring animation takes to settle. Useful for sizing <Sequence> durations dynamically.
import { measureSpring } from "remotion"; const duration = measureSpring({ fps: 30, config: { damping: 15, stiffness: 120 } }); // Returns number of frames until settled
Animation Math Available
Combine springs by adding/subtracting. Classic pattern for enter+exit: scale = enterSpring - exitSpring. The value goes 0 -> 1 -> 0.
const enter = spring({ frame, fps }); const exit = spring({ frame, fps, delay: durationInFrames - 20, durationInFrames: 20, }); const scale = enter - exit; // 0 -> 1 (enter) -> 0 (exit)

Media Components

Video, audio, images, and asset loading
<OffthreadVideo> Used
Renders video frame-by-frame during render (no browser video decoder needed). More reliable than <Html5Video> for production renders. Supports startFrom, volume, playbackRate.
<OffthreadVideo src={staticFile("footage.mov")} startFrom={300} // start at frame 300 volume={0} // mute (audio separate) />
<Audio> Used
Audio playback synced to Remotion's timeline. Supports trimming via startFrom, volume control (static or function), and playbackRate for speed changes.
<Audio src={staticFile("footage.mov")} startFrom={videoStartFrame} volume={1} /> // Dynamic volume (fade out): volume={(f) => interpolate(f, [700,750], [1,0], {extrapolateLeft:"clamp"}) }
<Img> Used
Renders an image and waits for it to load before rendering the frame. Prevents blank frames during render. Always use instead of raw <img>.
<Img src={staticFile("covers/mixtape.png")} style={{ width: 300, borderRadius: 16 }} />
staticFile() Used
References files in the public/ folder. Returns a URL that works in both preview and render. The only correct way to reference local assets.
import { staticFile } from "remotion"; staticFile("footage/dj-set.mov") // -> /public/footage/dj-set.mov (preview) // -> bundled URL (render)
<AnimatedImage> Available
Display GIFs, animated WebP, or animated AVIF synced to the timeline. Useful for animated stickers or reaction GIFs in clips.
<AnimatedImage src={staticFile("sticker.gif")} />
@remotion/media-utils Available
Get audio metadata, duration, and waveform data in JavaScript. getAudioData(), getVideoMetadata(), getWaveformPortion(). Could replace our Python audio analyzer.
import { getAudioData } from "@remotion/media-utils"; const data = await getAudioData(src); // { channelWaveforms, sampleRate, ... }

Transitions

Built-in scene transitions via @remotion/transitions
<TransitionSeries> Available
Like <Series> but with transitions between scenes. Each transition has a presentation (visual effect) and timing (duration/curve). Built-in presentations: fade(), slide(), wipe(), flip(), clockWipe(), iris(), cube().
TransitionSeries visual flow: |=====Scene A (40f)======| |=FADE 30f=| |========Scene B (60f)=========| |=WIPE 20f=| |=====Scene C (50f)======| Total duration = 40 + 60 + 50 - 30 - 20 = 100 frames Built-in presentations: fade() Crossfade opacity slide() Push previous scene out wipe() Slide over (8 directions) flip() 3D flip reveal clockWipe() Circular clock reveal iris() Center circle reveal
import { TransitionSeries, linearTiming } from "@remotion/transitions"; import { fade } from "@remotion/transitions/fade"; <TransitionSeries> <TransitionSeries.Sequence durationInFrames={90}> <HookScene /> </TransitionSeries.Sequence> <TransitionSeries.Transition presentation={fade()} timing={linearTiming({ durationInFrames: 30 })} /> <TransitionSeries.Sequence durationInFrames={150}> <DropScene /> </TransitionSeries.Sequence> </TransitionSeries>

Captions & Subtitles

TikTok-style word-by-word captions via @remotion/captions
@remotion/captions Available
Parse .srt subtitle files and create animated TikTok-style word-by-word captions. Works with Whisper.cpp for auto-transcription. Could add lyric overlays to DJ clips.
import { createTikTokStyleCaptions } from "@remotion/captions"; import { parseSrt } from "@remotion/captions"; const captions = parseSrt(srtFileContent); const { pages } = createTikTokStyleCaptions({ captions, combineTokensWithinMilliseconds: 1200, }); // Each page has: text, startMs, durationMs, tokens[] // Each token has: text, fromMs, toMs // -> Animate word highlights based on timing
PYB use case: Auto-caption DJ Gianni's TikTok clips with track titles or lyric snippets. Combine with Whisper.cpp for voice-over captions.

Fonts & Styling

Typography, Google Fonts, and design tokens
@remotion/google-fonts Used
Load Google Fonts reliably during render. Must call loadFont() at module level (not inside components). Fonts are bundled into the output.
import { loadFont } from "@remotion/google-fonts/BebasNeue"; import { loadFont as loadBody } from "@remotion/google-fonts/SpaceGrotesk"; // Call at module level (outside component) const { fontFamily: display } = loadFont(); const { fontFamily: body } = loadBody();
CSS-in-JS Styling Used
Remotion uses standard React inline styles. All CSS properties work including filter, mixBlendMode, clipPath, transform, textShadow, and SVG filters.
// Effects we use in release clips: filter: `brightness(${b}) contrast(1.1)` mixBlendMode: "overlay" // grain mixBlendMode: "screen" // light leaks clipPath: `inset(0 ${100-pct}% 0 0)` // wipe transform: `scale(${s}) translate(${x}px)`

Rendering & Output

CLI rendering, server-side rendering, and cloud deployment
Remotion Studio Used
Browser-based preview with timeline scrubbing, props editor, and real-time preview. Launch with npx remotion studio.
# Preview in browser with live reload $ npx remotion studio # Opens at http://localhost:3000 # Scrub timeline, edit props, preview effects
CLI Render Used
Render videos from the command line. Supports codec, quality (CRF), pixel format, and custom props via JSON. Our config: H.264, CRF 18, yuv420p.
$ npx remotion render src/index.ts ReleaseClip \ output.mp4 \ --props='{"genre":"afro",...}' # Our remotion.config.ts: codec: h264 crf: 18 (near-lossless) pixelFormat: yuv420p (TikTok compatible)
@remotion/renderer (SSR) Available
Render videos programmatically from Node.js/Bun. Bundle once, render many videos with different props. Could automate batch rendering for all mixtape volumes.
import { bundle } from "@remotion/bundler"; import { renderMedia, selectComposition } from "@remotion/renderer"; const bundleUrl = await bundle({ entryPoint }); const comp = await selectComposition({ serveUrl: bundleUrl, id: "ReleaseClip", inputProps: { genre: "afro", ... } }); await renderMedia({ composition: comp, serveUrl: bundleUrl, codec: "h264", outputLocation: "out/clip.mp4" });
@remotion/player Used
Embed a Remotion video in any React web app with play/pause controls. Could be used on the PYB website to preview video templates.
import { Player } from "@remotion/player"; <Player component={ReleaseClip} durationInFrames={750} fps={30} compositionWidth={1080} compositionHeight={1920} inputProps={clipProps} controls />
@remotion/lambda Available
Render videos on AWS Lambda. Massively parallel rendering for fast output. Pay-per-use. Could enable on-demand clip generation from a web form.
// Cloud rendering flow: 1. deploySite() // Upload bundle to S3 2. deployFunction() // Create Lambda 3. renderMediaOnLambda() // Trigger 4. getRenderProgress() // Poll 5. downloadMedia() // Get result

3D & Advanced

React Three Fiber integration and advanced patterns
@remotion/three Available
Full React Three Fiber integration. Use <ThreeCanvas> for 3D scenes synced to the timeline. Supports video-as-texture for 3D phone mockups, logo reveals, particle systems, and more.
import { ThreeCanvas } from "@remotion/three"; <ThreeCanvas width={1080} height={1920}> <ambientLight /> <mesh rotation={[frame * 0.02, frame * 0.03, 0]} scale={spring({ frame, fps })} > <boxGeometry /> <meshStandardMaterial /> </mesh> </ThreeCanvas>
delayRender() Available
Block rendering until async operations complete (API calls, image loading, font loading). Call continueRender() when ready. Prevents blank/incomplete frames.
const [handle] = useState(() => delayRender("Loading data") ); useEffect(() => { fetchData().then(() => { continueRender(handle); }); }, []);
getInputProps() Available
Read props passed via CLI --props or Player inputProps. Our template already uses defaultProps but this enables runtime overrides for dynamic content.
const props = getInputProps(); // From CLI: --props='{"genre":"afro"}' // Returns: { genre: "afro" }
