# Conversion Rate Optimization (CRO)

## When to Use

- Auditing a landing page for conversion performance
- Designing or improving a landing page layout, copy, or user flow
- Setting up, analyzing, or interpreting A/B tests
- Optimizing web forms (lead gen, signup, contact)
- Reducing cart abandonment or improving checkout completion
- Diagnosing why a page or funnel has a low conversion rate

---

## Landing Page Audits

- **5-second test**: Does the page communicate its value proposition within 5 seconds?
- **Above-the-fold analysis**: Headline clarity, subheadline support, hero image relevance, primary CTA visibility
- **Trust signal inventory**: Logos, testimonials, reviews, certifications, security badges, guarantees
- **CTA assessment**: Clarity, contrast, placement, copy specificity, number of competing CTAs
- **Form evaluation**: Field count, field labels, required vs optional, error handling, multi-step vs single-step
- **Page speed impact**: Load time correlation to bounce rate, Core Web Vitals as conversion factors
- **Mobile experience**: Touch targets, scroll depth, thumb-zone CTA placement, mobile-specific friction points
- **Content hierarchy**: Visual hierarchy, F-pattern or Z-pattern scanning support
- **Objection handling**: Whether the page addresses common objections before the conversion point

## A/B Testing Framework

- **ICE prioritization**: Score potential tests by Impact (1-10), Confidence (1-10), and Ease (1-10)
- **Hypothesis format**: "If we [change], then [metric] will [improve/decrease] because [rationale]"
- **Sample size calculation**: Based on baseline conversion rate, minimum detectable effect, statistical power (80%), significance level (95%)
- **Test duration**: Minimum 1-2 weeks to capture weekly patterns
- **Result interpretation**: Statistical significance, practical significance, segment analysis, revenue impact

## Form Optimization

- **Field reduction**: Each additional field reduces conversion by approximately 4-7%
- **Progressive profiling**: Collect information across multiple interactions
- **Inline validation**: Real-time feedback reduces form abandonment
- **Smart defaults**: Pre-fill known data, use sensible defaults, auto-complete
- **Multi-step forms**: Break long forms into logical steps with progress indicators

## Landing Page Audit Checklist (55 Items)

### Scoring Rubric - Landing Page Score

Each section scored 1-5. Weighted total produces Landing Page Score (0-100).

| Section | Weight | 1 (Poor) | 3 (Adequate) | 5 (Excellent) |
|---|---|---|---|---|
| Above-Fold Clarity | 20% | No clear value prop | Value prop present but generic | Specific, compelling value prop immediately visible |
| Trust & Credibility | 15% | No social proof | Some testimonials or logos | Layered trust signals matched to buyer stage |
| Copy & Messaging | 15% | Feature-focused, jargon-heavy | Benefit-oriented but lacks specificity | Customer-voice copy, objection handling, clear hierarchy |
| CTA & Form Design | 20% | Vague CTA, excessive fields | Clear CTA with reasonable form | High-contrast CTA, minimal friction, value-framed label |
| Visual Design | 10% | Cluttered, inconsistent | Clean layout with adequate hierarchy | Purposeful design directing eye flow to CTA |
| Mobile Experience | 10% | Broken or unusable on mobile | Functional but not optimized | Thumb-friendly, fast, native-feeling experience |
| Page Speed | 10% | >5s load time | 2-4s load time | <2s load time with optimized LCP/CLS |

**Score Interpretation:**
- 85-100: High-performing - test incremental improvements
- 65-84: Solid foundation - address weak sections
- 40-64: Underperforming - structural changes needed
- Below 40: Fundamental rebuild recommended

### Above-Fold Clarity (Items 1-10)
- [ ] 1. Headline communicates specific outcome or transformation
- [ ] 2. Subheadline adds context without repeating headline
- [ ] 3. Visitor can identify what you do, who it's for, and why it matters within 5 seconds
- [ ] 4. Hero image or visual reinforces the value proposition
- [ ] 5. Primary CTA is visible without scrolling on all viewports
- [ ] 6. Navigation is removed or minimized to reduce exit paths
- [ ] 7. Message matches the ad/email/referral source
- [ ] 8. No competing offers or distractions above the fold
- [ ] 9. Page title and meta description align with on-page messaging
- [ ] 10. Unique value differentiator is explicit

### Trust & Credibility (Items 11-18)
- [ ] 11. Customer logos relevant to target segment
- [ ] 12. Testimonials include full names, titles, company, and headshots
- [ ] 13. Quantified results in testimonials or case study snippets
- [ ] 14. Third-party validation (awards, certifications, press mentions)
- [ ] 15. Security badges near form or payment fields
- [ ] 16. Privacy language near data-collection points
- [ ] 17. Social proof is specific to the use case
- [ ] 18. Guarantee or risk-reversal statement is present

### Copy & Messaging (Items 19-28)
- [ ] 19. Copy leads with customer problems or desired outcomes
- [ ] 20. Benefits are specific and quantified where possible
- [ ] 21. Objections addressed on-page
- [ ] 22. Copy uses the audience's own language
- [ ] 23. Scannable formatting - short paragraphs, bullets, bold key phrases
- [ ] 24. No jargon or acronyms without explanation
- [ ] 25. Urgency or scarcity is genuine, not manufactured
- [ ] 26. Microcopy on buttons reduces anxiety
- [ ] 27. Content hierarchy follows awareness level of traffic source
- [ ] 28. FAQ addresses top 3-5 buying objections

### CTA & Form Design (Items 29-38)
- [ ] 29. CTA button text describes value received
- [ ] 30. CTA button has sufficient color contrast (4.5:1 ratio)
- [ ] 31. Only one primary CTA per viewport
- [ ] 32. Form requests only information needed for next step
- [ ] 33. Form field count matches offer value
- [ ] 34. Required vs optional fields clearly distinguished
- [ ] 35. Inline validation provides real-time feedback
- [ ] 36. Error messages are specific and helpful
- [ ] 37. Thank-you page continues the conversion journey
- [ ] 38. CTA is repeated at logical scroll points

### Visual Design (Items 39-44)
- [ ] 39. Visual hierarchy guides eye to CTA
- [ ] 40. Whitespace groups related elements
- [ ] 41. Imagery shows product in use or outcome achieved
- [ ] 42. Accent color reserved for CTAs
- [ ] 43. Typography readable (16px+ body, 1.5+ line height)
- [ ] 44. Directional cues point toward CTA

### Mobile Experience (Items 45-49)
- [ ] 45. Tap targets minimum 44x44px
- [ ] 46. Forms use appropriate input types for mobile keyboards
- [ ] 47. Sticky CTA on long mobile pages
- [ ] 48. Content not hidden behind toggles
- [ ] 49. Images responsive, no horizontal scrolling

### Page Speed & Technical (Items 50-55)
- [ ] 50. LCP under 2.5 seconds
- [ ] 51. CLS under 0.1
- [ ] 52. FID/INP under 200ms
- [ ] 53. Images compressed (WebP/AVIF)
- [ ] 54. Third-party scripts deferred
- [ ] 55. Tracking functional (analytics, heatmaps, conversion pixels)

## Common Conversion Killers - Ranked by Impact

| Rank | Conversion Killer | Avg. Impact on CVR | Frequency |
|---|---|---|---|
| 1 | Message mismatch between ad and landing page | -30 to -60% | Very common |
| 2 | Slow page load (>4s) | -20 to -40% | Common |
| 3 | Unclear or missing value proposition | -25 to -50% | Very common |
| 4 | Too many form fields for the offer value | -15 to -30% | Common |
| 5 | No trust signals near conversion point | -10 to -25% | Common |
| 6 | Competing navigation or exit paths | -10 to -20% | Very common |
| 7 | Generic CTA text ("Submit", "Click Here") | -5 to -15% | Extremely common |
| 8 | Broken mobile experience | -20 to -50% | Common |
| 9 | No objection handling on page | -10 to -20% | Common |
| 10 | Poor visual hierarchy - CTA not prominent | -5 to -15% | Common |

## Heuristic Evaluation Frameworks

### Cialdini's Principles Applied to Landing Pages

| Principle | Landing Page Application | Audit Question |
|---|---|---|
| Reciprocity | Offer value before asking (free tool, guide) | Does the page give something before requesting info? |
| Social Proof | Customer count, testimonials, logos | Is social proof specific, relevant, and layered? |
| Authority | Expert credentials, media mentions | Does the page establish expertise? |
| Scarcity | Limited availability, time-bound offers | Is scarcity genuine and verifiable? |
| Consistency | Micro-commitments before main ask | Does the page use progressive engagement? |
| Liking | Relatable language, founder story | Does the page feel human and approachable? |

### Fogg Behavior Model (B = MAP)

| Element | Audit Focus | Red Flags |
|---|---|---|
| **Motivation** | Does the page amplify desire and reduce fear? | Features over outcomes; no emotional hooks; no risk reversal |
| **Ability** | Is the action as simple as possible? | Too many fields; unclear next step; cognitive overload |
| **Prompt** | Is the CTA visible, clear, and timed correctly? | CTA below fold; vague button text; no CTA repetition |
